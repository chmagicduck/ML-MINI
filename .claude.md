## 项目概览

### 核心信息

- **项目名称**：吗喽薪事 (Malou Slacker)
- **项目类型**：微信小程序（WeChat Mini Program）
- **核心功能**：实时计算"带薪摸鱼"收益，展示工作时间转化的金钱价值
- **目标用户**：追求职场 WLB、对薪资敏感的 90/00 后白领
- **主要语言**：TypeScript + WeChat WXML/WXSS
- **技术栈**：原生小程序 API（无框架依赖）

### 核心特点

✅ **实时计算**：100ms 刷新频率，金额实时跳动
✅ **后台计时**：时间戳差值法，杀进程后自动补齐
✅ **节假日智能**：内置 2026 年法定节假日和调休
✅ **等级激励**：5 个等级系统，升级时触发视觉反馈
✅ **离线收益**：冷启动时显示离线期间赚取的金额
✅ **分享裂变**：一键生成摸鱼战报海报

## 关键业务逻辑

### 1. 收益计算公式

```typescript
// 基础公式
秒薪 = 月薪 ÷ 当月工作日数 ÷ (每日工作秒数)
今日摸鱼收益 = 秒薪 × 摸鱼秒数
今日入账工资 = 秒薪 × 已上班秒数 (不受摸鱼影响)
本月已赚 = 已过工作日薪资 + 今日入账工资
```

**关键约束**：
- 秒精度刷新（100ms 视觉更新）
- 避免分钟精度导致更新卡顿
- 今日入账工资下班后锁定为日薪

### 2. 后台计时逻辑（时间戳差值法）

**目的**：杀进程后重启，收益自动同步最新差值

**实现**：
```typescript
// 开始摸鱼
wx.setStorageSync('moyu_start_time', Date.now())

// 页面 onShow 时
const diff = Date.now() - wx.getStorageSync('moyu_start_time')
const newSeconds = diff / 1000
```

**注意**：确保 storage key 一致，避免覆盖

### 3. 摸鱼状态保持

**场景 A**：切换子页面（JS 上下文存活）
```typescript
// onHide
_lastHideWorkedSecs = calcTodayWorkedSeconds(settings)

// onShow (若 isSlacking)
const catchUpSecs = getTodayWorkedSeconds() - _lastHideWorkedSecs
slackingSeconds += catchUpSecs
```

**场景 B**：冷启动（杀进程后）
- 不补齐，`isSlacking` 默认 false，直接待机

**关键**：补齐量 = 上班已过时间差，绝不补非工作时间

### 4. 等级升级触发

**时机**：
- 首页 timer tick 中检查累计收益是否突破阈值
- `newLevel.threshold > _lastLevel.threshold` 时触发

**流程**：
```typescript
const newLevel = getMoyuLevel(totalMoney)
if (newLevel.threshold !== _lastLevelThreshold && isSlacking) {
  // 存储待处理升级事件
  setPendingLevelUp(newLevel)
  // 触发升级弹窗
  wx.showModal({ ... })
  // 振动反馈
  wx.vibrateShort()
  // 触发金币雨
  drawCoinRain()
  _lastLevelThreshold = newLevel.threshold
}
```

**避免**：杀进程后重启不应重复弹窗（使用 `_levelUpShown` 标记）

### 5. 工作时间限制

**规则**：
- 仅在 `[workStartTime, workEndTime]` 时间段允许开启摸鱼
- 午休期间 (`[lunchStart, lunchEnd]`) **允许**开启
- 工作时间外提示："工作时间外无法开启摸鱼 🐟"

**实现**：
```typescript
if (!isWorkingNow(settings)) {
  wx.showToast({ title: '工作时间外无法开启摸鱼 🐟' })
  return
}
```

### 6. 离线收益弹窗

**触发条件**（全部满足）：
- 冷启动（非 onShow 恢复）
- 距上次退出 > 1 小时
- 同一天（未跨天）
- 当前在工作时间

**实现**：
```typescript
const lastExitTime = wx.getStorageSync('last_exit_time')
const timeDiff = Date.now() - lastExitTime
if (timeDiff > 3600000 && isSameDay && isWorkingNow()) {
  const offlineEarnings = calcOfflineEarnings(...)
  showOfflineEarningsModal(offlineEarnings)
}
```

## 工程最佳实践

### 代码组织

**✅ 应该做**：
- 创建小的、高内聚的工具函数（<50 行）
- 将计算逻辑封装在 `calculator.ts`

- 修改原对象，应返回新对象
- 硬编码常数（应定义在 `types.ts` 中）

### 数据流规范

```typescript
// ❌ 错误：直接修改 data
this.data.totalMoney = 100
this.setData({ totalMoney: 100 })

// ✅ 正确：计算后更新
const newTotal = calcMonthEarningsV2(settings)
this.setData({ monthEarnings: formatMoney(newTotal) })
```

### 时间处理

```typescript
// ❌ 错误：分钟精度，导致本月已赚更新卡顿
const minutes = Math.floor(seconds / 60)

```

## 常见修复指南

### 问题 1：金额显示不实时更新

**症状**：本月已赚数字 1 分钟才跳一次
**原因**：计算中使用了分钟精度
**修复**：在 `calculator.ts` 中改用秒精度

```typescript
// 之前
const minutes = getMinutes()
const earningsPerMin = ...

// 之后
const seconds = getTodayWorkedSeconds(settings)
const earningsPerSec = ...
```

### 问题 2：杀进程后收益丢失

**症状**：后台被杀，重启小程序后摸鱼秒数重置
**原因**：仅在内存中记录秒数，未持久化
**修复**：每次摸鱼结束时调用 `commitMoyuSession()`

```typescript
_pause() {
  const seconds = getTodaySlackingSeconds()
  commitMoyuSession(seconds)  // 持久化
  this.setData({ isSlacking: false })
}
```

### 问题 3：战报页和首页数据不一致

**症状**：首页摸鱼收益 ¥50，战报页看不到
**原因**：战报页仅显示已提交的会话，未计入进行中会话
**修复**：战报页 `onShow` 时额外读取实时收益

```typescript
const committedMoney = getStats().total_moyu_money
const liveEarnings = getLiveEarnings(settings)
const totalDisplayed = committedMoney + liveEarnings
```

### 问题 4：午休时间计算错误

**症状**：中午吃饭时间被算成了摸鱼
**原因**：未从工作时间中扣除午休
**修复**：在 `getTodayWorkedSeconds()` 中扣除午休

```typescript
let workedSeconds = (currentTime - workStartTime) / 1000
if (currentTime > lunchEnd) {
  workedSeconds -= (lunchEnd - lunchStart) / 1000
}
```

### 问题 5：等级升级弹窗重复出现

**症状**：从战报页返回首页，升级弹窗重复弹出
**原因**：每次 `onShow` 都检查升级，未标记为已显示
**修复**：使用 `_levelUpShown` 标记或 `clearPendingLevelUp()`

```typescript
const pending = getPendingLevelUp()
if (pending && !_levelUpShown) {
  wx.showModal({ ... })
  _levelUpShown = true
  clearPendingLevelUp()
}
```

## 关键时间点

### 默认工作时间

- **上班**：09:00
- **午休**：12:00 - 13:00（1 小时）
- **下班**：18:00
- **每日工作秒数**：(18:00 - 09:00 - 1 小时) × 3600 = 28,800 秒

### 发薪日周期

- 默认：每月 10 号
- 用户可自定义 1-31 号
- 跨月份时自动调整

### 计时器刷新率

- **金额显示**：100ms 更新一次
- **本月工作日数**：每天 onShow 时更新（不需放入 timer）
- **月份变化**：凌晨 0:00 自动切换

## 性能优化

### 计时器

```typescript
// ❌ 频繁创建新 timer
setInterval(() => this._refresh(), 100)

// ✅ 保存 timer ID，便于清理
this._timerId = setInterval(() => this._refresh(), 100)
this.onUnload = () => clearInterval(this._timerId)
```

- 摸鱼热力图使用 Canvas（避免大量 DOM 元素）
