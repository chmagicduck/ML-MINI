## 一、产品定义与战略

### 1.1 核心定位
- **产品名称**：吗喽薪事（Malou Slacker）
- **核心定位**：一款专为打工人设计的"反卷"可视化工具
- **核心价值**：通过实时计算"带薪摸鱼"收益，缓解职场焦虑，提供情绪价值
- **目标用户**：追求职场 WLB（工作生活平衡）、喜欢玩梗、对薪资结构敏感的 90/00 后职场人
- **Slogan**：只要我不努力，老板就永远过不上想要的生活

### 1.2 品牌基因

**视觉风格**：
- **主色调**：护眼绿 (#E8F5E9)
- **辅助色**：金色 (#FFD700)，用于高级等级标记
- **设计语言**：反差萌。用最严谨的金融图表风格，展示最荒诞的"摸鱼数据"
- **核心元素**：吗喽（猴子）Emoji/插画、实时跳动的数字、极简主义卡片
- **UI 框架**：Soft UI（温和的卡片投影）

### 1.3 商业闭环
- **社交激励**：生成"摸鱼战报"分享朋友圈（裂变）
- **成就系统**：解锁等级勋章，激励持续使用
- **功能扩展**：可接入"老板来了"紧急避险模式、广告、皮肤系统

## 二、核心业务逻辑

### 2.1 收益计算公式

```
日薪 D = 月薪 / 当月工作日数

秒薪 S = D / (下班时间 - 上班时间 - 午休时长) / 3600

今日摸鱼收益 = 秒薪 × 累计摸鱼秒数

今日入账工资 = 秒薪 × 今日已上班秒数（纯基于时间流逝）
```

### 2.2 时间戳差值法（后台计时）

**解决问题**：小程序被杀后进程重启，摸鱼收益自动同步

**流程**：
1. 用户点击"开始摸鱼"，`wx.setStorageSync('moyu_start_time', Date.now())`
2. 页面 `onShow` 时，计算差值 `diff = Date.now() - moyu_start_time`
3. 即使小程序被杀，只要缓存存在，重新打开时收益自动同步最新差值
4. 收益公式：`今日收益 = (diff / 1000) × 秒薪`

### 2.3 本地节假日引擎

- 在 `utils/holiday.ts` 中内置当年法定节假日和调休数据
- 自动判断今日是否为"法定加班日"或"带薪节假日"
- 若为周末或节假日，首页显示"休息中，目前是纯利润时间"

### 2.4 等级系统

| 等级 | 阈值 | 描述 |
|-----|-----|-----|
| 职场牛马 | < ¥500 | 刚踏上摸鱼之路 |
| 摸鱼学徒 | ¥500 - ¥1,999 | 初窥摸鱼门道 |
| 带薪锦鲤 | ¥2,000 - ¥9,999 | 进入高端摸鱼行列 |
| 划水宗师 | ¥10,000 - ¥49,999 | 上班是副业，摸鱼是正业 |
| 摸鱼大圣 | ¥50,000+ | 月薪的敌人已登陆（金色边框） |

### 4.2 核心功能

**计时器交互**：
- 点击中央圆圈开启/关闭"摸鱼模式"
- 开启后数字实时跳动（100ms 刷新率）
- 所有金额卡片永久应用脉冲动画（value-pulse）

**工作时间限制**：
- 仅在 `[workStartTime, workEndTime]` 时间段允许开启摸鱼
- 午休期间允许开启
- 工作时间外提示："工作时间外无法开启摸鱼 🐟"

**离线收益弹窗**：
- 触发条件：冷启动 + 距上次退出 > 1 小时 + 同一天 + 当前在工作时间
- 文案示例：
  ```
  🐒 你不在的时候...
  我偷偷帮你从老板兜里掏走了 ¥XX.XX
  上班打卡，继续薅！
  ```

**摸鱼状态保持**：
- 切换子页面时记录当前上班秒数
- 返回时计算差值并补齐摸鱼秒数
- 确保离线期间不补非工作时间

### 4.3 特效动画

- **金币雨**：每跨 ¥10 整数门槛触发一次；等级升级时触发一次（15 个金币粒子，1.5 秒落下）
- **圆环呼吸**：摸鱼中时，2 秒周期 box-shadow 呼吸光晕（绿色 rgba(76,175,80,0.2)）
- **数字脉冲**：永久应用，传递"钱一直在流动"的爽感
- **振动反馈**：点击和升级时可选 `wx.vibrateShort`（用户可配置开关）

### 4.4 计算说明卡片

**位置**：首页顶部（仪表盘下方），可折叠

**内容**：
```
📐 这些数字怎么算的？

• 秒薪 = 月薪 ÷ 本月工作日数 ÷ 每日工作秒数
• 今日入账工资 = 秒薪 × 今日已上班秒数
• 今日摸鱼收入 = 秒薪 × 摸鱼计时秒数
• 本月已赚 = 已过工作日薪资 + 今日入账工资

当前秒薪：¥0.0048/秒（根据你的设置实时计算）
```

### 12.2 代码风格约定

- **不可变性**：操作数据时创建新对象，不修改原对象
- **函数长度**：单个函数不超过 50 行
- **文件大小**：单个页面不超过 800 行
- **嵌套深度**：不超过 4 层
- **注释规范**：仅在逻辑不自明时添加，避免过度注释
- **错误处理**：必须全面处理异常，提供友好错误提示
- **输入验证**：验证所有用户输入和外部数据
- **无硬编码**：配置值存储在常量或 settings 中

## 十五、关键名词解释

| 词汇 | 定义 |
|-----|-----|
| **摸鱼** | 在工作时间内进行与工作无关的活动 |
| **带薪拉屎** | 在工作时间内上厕所（特殊的摸鱼类型） |

| **等级路线图** | 展示所有等级及解锁条件的可视化流程 |
| **时间戳差值法** | 通过记录起始时刻和当前时刻的差值来计算经过时间的方法 |
| **工作日模式** | 用户配置的工作日规则（双休/大小周等） |
