### 修复3：摸鱼全局进度同步（升级弹窗）
**问题**：

**代码变更**：

## 一、核心改进详解

### 改进 1：等级系统重设（+激励体验）

**问题**：月薪 8000-10000 的用户，从 ¥0 升到首个等级需要 1 天，但只有 5 级，容易产生"升级完没有目标"的感受。

**新方案**：从 5 级升至 8 级，重新调整阈值

| 等级 | 原阈值 | 新阈值 | 变化 | 称号 | 文案 |
|-----|--------|--------|------|------|------|
| Lv1 | ¥500 | ¥100 | ↓ 80% | 职场牛马 | 刚踏上摸鱼之路，革命尚未成功 |
| Lv2 | ¥500-2000 | ¥100-300 | ↓ 70% | 摸鱼学徒 | 初窥摸鱼门道，但距大师还远 |
| Lv3 | ¥2000-10000 | ¥300-800 | 新增 | 小有成就 | 已掌握摸鱼精髓，开始进阶 |
| Lv4 | - | ¥800-2000 | 新增 | 摸鱼学者 | 进入高端摸鱼行列，薪水不再是梦 |
| Lv5 | - | ¥2000-5000 | 新增 | 带薪锦鲤 | 我这辈子唯一的坚持，就是每天带薪拉屎 |
| Lv6 | - | ¥5000-15000 | 新增 | 成熟摸鱼手 | 上班是我的副业，摸鱼才是正业 |
| Lv7 | ¥10000-50000 | ¥15000-40000 | 新增 | 划水宗师 | 老板的工资单，是我的致富密码 |
| Lv8 | ¥50000+ | ¥40000+ | 新增 | 摸鱼大圣 | 老板的钱，最终都要回到打工人的口袋 |

**等级配色**：
- Lv1-2：灰色（入门阶段）
- Lv3-4：绿色（成长阶段）
- Lv5-6：金色（成熟阶段）
- Lv7：蓝色（大师阶段）
- Lv8：红金色（传说阶段，带边框闪烁）

**影响**：用户升级速度提升，新用户在第 2-3 天就能体验 3-4 次升级，大幅提升留存。

#### 3.2 仪表盘文案优化

**从**：`摸鱼占比 x%`

**改为**：`还有 X 小时 Y 分钟下班` + 剩余时间进度条

**实现**：
```typescript
// pages/index/index.ts
data() {
  return {
    timeUntilOffWork: '还有 8 小时 30 分钟下班',
    offWorkProgress: 15,  // 0-100 的百分比（已工作 2% → 还有 85% 要工作）
  }
}

_updateTimeUntilOffWork() {
  const now = new Date()
  const [workStartHour, workStartMin] = this.data.settings.work_time[0].split(':').map(Number)
  const [workEndHour, workEndMin] = this.data.settings.work_time[1].split(':').map(Number)

  const workEndTime = new Date()
  workEndTime.setHours(workEndHour, workEndMin, 0)

  if (now >= workEndTime) {
    this.setData({
      timeUntilOffWork: '已下班，尽情摸鱼！',
      offWorkProgress: 100,
    })
    return
  }

  const msRemaining = workEndTime.getTime() - now.getTime()
  const hoursRemaining = Math.floor(msRemaining / 3600000)
  const minsRemaining = Math.floor((msRemaining % 3600000) / 60000)

  const msTotal = workEndTime.getTime() - new Date(new Date().setHours(workStartHour, workStartMin, 0)).getTime()
  const progress = 100 - Math.round((msRemaining / msTotal) * 100)

  this.setData({
    timeUntilOffWork: `还有 ${hoursRemaining} 小时 ${minsRemaining} 分钟下班`,
    offWorkProgress: progress,
  })
}
```

### 改进 4：会议烧钱机自定义人均月薪修复（BUG 修复）

**问题**：自定义人均月薪 Toggle 按钮点击没反应

**根本原因**：可能是以下几种情况：
1. Toggle 组件未绑定 `bindchange` 事件
2. data 字段名不一致
3. 按钮被 disabled 属性锁定

**修复方案**：

```typescript
// pages/meeting/index.ts
data() {
  return {
    useCustomSalary: false,        // Toggle 状态
    customSalaryPerPerson: 10000,  // 自定义月薪
    participantCount: 1,
    meetingCost: '¥0.00',
    meetingStartTime: null,
  }
}

onToggleCustomSalary(e) {
  this.setData({
    useCustomSalary: e.detail.value
  })
}

onCustomSalaryChange(e) {
  const salary = Number(e.detail)
  if (salary > 0) {
    this.setData({
      customSalaryPerPerson: salary
    })
  }
}

_calculateMeetingCost() {
  const settings = getSettings()
  const salary = this.data.useCustomSalary
    ? this.data.customSalaryPerPerson
    : settings.salary

  const secondSalary = salary / getWorkingDaysInMonth(...) / 28800
  const elapsedSeconds = (Date.now() - this.data.meetingStartTime) / 1000
  const cost = secondSalary * elapsedSeconds * this.data.participantCount

  this.setData({
    meetingCost: formatMoney(cost)
  })
}
```

### 改进 5：等级升级弹窗修复（BUG 修复）

**问题**：升级了但弹窗没弹出

**根本原因**（诊断）：
1. 升级检测逻辑未执行（timer 未启动）
2. 升级条件判断错误（如 `if (newLevel.threshold > _lastLevel.threshold)` 写反了）
3. 弹窗检测条件过严（如要求 `isSlacking === true`，但用户在战报页看升级）
4. 待处理升级事件未被正确读取或显示

**修复方案**：

```typescript
// pages/index/index.ts
data() {
  return {

    currentLevel: {},

_levelUpShown = false
_timerId = null

onShow() {
  // 检查待处理升级（可能是从其他页面返回时的升级）

  if (this._timerId) clearInterval(this._timerId)

    const settings = getSettings()
```

```typescript
    const offlineSeconds = (Date.now() - runningState.shit_last_hide_time) / 1000
```

### 改进 7：摸鱼热力图优化（近 30 天维度）

**问题**：
- 当前为整月展示，第一天的数据看不清
- 第二天还没有看到染色框

**新方案**：改为"近 7 天"展示，更快获得反馈

```typescript
// pages/stats/index.ts

data() {
  return {
    heatmapDays: [],  // 数组，包含最近 7 天
  }
}

onShow() {
  this._renderHeatmap()
}

_renderHeatmap() {
  const today = new Date()
  const days = []

  // 生成最近 7 天
  for (let i = 6; i >= 0; i--) {
    const date = new Date(today)
    date.setDate(date.getDate() - i)
    const dateStr = this._formatDate(date)

    const stats = getStats()
    const seconds = stats.moyu_days_map[dateStr] || 0
    const earnings = seconds * getSecondSalary(getSettings())

    // 颜色深度：基于收益
    // 0 → 无色
    // 1-100 → 浅绿
    // 100-500 → 中绿
    // 500+ → 深绿
    const intensity = seconds > 0 ? Math.min(255, Math.floor(255 * earnings / 500)) : 0

    days.push({
      date: dateStr,
      dayOfWeek: this._getDayOfWeek(date),
      seconds,
      earnings: formatMoney(earnings),
      color: `rgba(76, 175, 80, ${intensity / 255})`,
      intensity,  // 0-255，用于 Canvas 绘制
    })
  }

  this.setData({ heatmapDays: days })

  // 使用 Canvas 绘制
  setTimeout(() => {
    this._drawHeatmapOnCanvas()
  }, 100)
}

_drawHeatmapOnCanvas() {
  const ctx = wx.createCanvasContext('heatmapCanvas')
  const cellSize = 40
  const padding = 10

  this.data.heatmapDays.forEach((day, index) => {
    const x = padding + index * (cellSize + 5)
    const y = padding

    // 绘制底色
    ctx.fillStyle = '#f0f0f0'
    ctx.fillRect(x, y, cellSize, cellSize)

    // 绘制热力颜色
    if (day.intensity > 0) {
      ctx.fillStyle = day.color
      ctx.fillRect(x, y, cellSize, cellSize)
    }

    // 绘制边框
    ctx.strokeStyle = '#ddd'
    ctx.lineWidth = 1
    ctx.strokeRect(x, y, cellSize, cellSize)

    // 绘制日期标签
    ctx.fillStyle = '#666'
    ctx.font = '12px Arial'
    ctx.textAlign = 'center'
    ctx.fillText(day.dayOfWeek, x + cellSize / 2, y + cellSize + 15)
  })

  ctx.draw()
}

_formatDate(date) {
  const year = date.getFullYear()
  const month = String(date.getMonth() + 1).padStart(2, '0')
  const day = String(date.getDate()).padStart(2, '0')
  return `${year}-${month}-${day}`
}

_getDayOfWeek(date) {
  const days = ['日', '一', '二', '三', '四', '五', '六']
  return days[date.getDay()]
}
```
