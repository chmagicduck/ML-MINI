**涉及文件**：

## 问题 2：重置数据后薪资信息未重置

### 现象
1. 点击重置所有数据后
2. 数据似乎被清除了，但首页仍显示旧的月薪/薪资信息
3. 战报页仍显示旧的总累计收益

### 根本原因
`resetAllData()` 方法在 `pages/settings/basic.ts` 中只清除了部分关键 Storage 项：
- ✅ `moyuStats`
- ✅ `poopStats`
- ❌ **未清除** `userSettings`（包含月薪信息）
- ❌ **未清除** `poopRunningState`、`pendingLevelUp` 等

### 修复方案
改进 `resetAllData()` 方法，**完整清除所有存储项**：

```typescript
// 旧（不完整）
resetAllData() {
  wx.showModal({
    title: '确认清除？',
    content: '将删除所有摸鱼记录，此操作不可恢复',
    success: (res) => {
      if (res.confirm) {
        wx.setStorageSync('moyuStats', {})
        wx.setStorageSync('poopStats', {})
        wx.setStorageSync('moyu_running_state', {})
        wx.reLaunch({ url: '/pages/index/index' })
      }
    }
  })
}

// 新（完整）
resetAllData() {
  wx.showModal({
    title: '确认清除？',
    content: '将删除所有摸鱼记录和设置，此操作不可恢复',
    confirmColor: '#EF5350',
    success: (res) => {
      if (res.confirm) {
        // 获取所有 Storage 的 keys
        const storageInfo = wx.getStorageInfoSync()

        // 逐个删除关键项
        const keysToRemove = [
          'userSettings',        // ✅ 清除薪资设置
          'moyuStats',          // ✅ 清除摸鱼累计统计
          'poopStats',          // ✅ 清除拉粑粑统计
          'poopRunningState',   // ✅ 清除拉粑粑运行状态
          'pendingLevelUp',     // ✅ 清除待处理升级
          'lastExitState',      // ✅ 清除离线收益状态
          'initialIdentityShown', // ✅ 清除初始身份标记
        ]

        // 清除所有 slackingToday_* 项
        if (storageInfo.keys) {
          storageInfo.keys.forEach(key => {
            if (key.startsWith('slackingToday_')) {
              wx.removeStorageSync(key)
            }
          })
        }

        // 清除关键项
        keysToRemove.forEach(key => {
          wx.removeStorageSync(key)
        })

        wx.showToast({ title: '已清除所有数据', icon: 'success' })

        // 延迟返回首页（让清除动画显示）
        setTimeout(() => {
          wx.reLaunch({ url: '/pages/index/index' })
        }, 1000)
      }
    }
  })
}
```

**涉及文件**：
- `pages/settings/basic.ts` - 改进 `resetAllData()` 方法（删除后将不需要此方法）
- `utils/storage.ts` - 无需改动（存储函数已完整）

## 问题 3：开启摸鱼后跳到战报页，数据不实时更新

### 现象
1. 首页开启摸鱼计时（计时器正常运行，显示正常）
2. 点击底部 Tab 切换到战报页（stats）
3. **战报页**的：
   - 圆形进度条（workedPercent）不变化
   - 总累计收益显示有延迟
   - 下班倒计时不更新

### 根本原因
战报页（`pages/stats/index.ts`）的 `onShow()` 只在初次进入时加载数据：

```typescript
onShow() {
  const quote = MOYU_QUOTES[Math.floor(Math.random() * MOYU_QUOTES.length)]
  this.setData({ quote })
  this._loadStats()  // ← 仅加载一次

  // ... 升级通知检查 ...
}
```

- 当用户在首页摸鱼时，战报页的数据是**静态的**，不会每秒更新
- 虽然首页每 100ms 更新一次，战报页却是冻结状态

### 修复方案
在战报页添加**始终运行的时钟**，每秒刷新数据（类似首页的 `_startClock()`）：

```typescript
// 添加模块级变量
let _statsClock: ReturnType<typeof setInterval> | null = null

Page({
  data: {
    // ... 现有数据 ...
  },

  onShow() {
    const quote = MOYU_QUOTES[Math.floor(Math.random() * MOYU_QUOTES.length)]
    this.setData({ quote })
    this._loadStats()

    // V1.0.1 Fix: 启动实时时钟
    this._startStatsClock()

    // ... 升级通知检查 ...
  },

  onHide() {
    this._stopStatsClock()
  },

  onUnload() {
    this._stopStatsClock()
  },

  // ──── 实时时钟（每秒更新）────
  _startStatsClock() {
    if (_statsClock !== null) return
    _statsClock = setInterval(() => {
      this._loadStats()  // 每秒重新加载数据
    }, 1000)
  },

  _stopStatsClock() {
    if (_statsClock !== null) {
      clearInterval(_statsClock)
      _statsClock = null
    }
  },

  _loadStats() {
    // 现有的 _loadStats 逻辑不需改动
    const stats = getMoyuStats()
    // ... 计算显示 ...
  },
})
```

**涉及文件**：
- `pages/stats/index.ts` - 添加 `_startStatsClock()`、`_stopStatsClock()` 方法

**性能考量**：
- ✅ 每秒一次足够满足用户体验
- ✅ 不会占用过多 CPU（计算量小）
- ✅ 离开页面时自动停止

## 问题 4：升级弹窗仅首次显示，后续升级无反应

### 现象
1. 首次开启摸鱼时，显示"获得初始身份"弹窗 ✅
2. 继续摸鱼，当累计收益达到 ¥300（升至 Lv2）时
   - **弹窗未弹出** ❌
   - 但首页仍可见等级变化
3. 返回首页后仍无升级弹窗

### 根本原因
首页（`pages/index/index.ts`）的升级检测存在多个问题：

1. **初始化不当**：`_lastLevelThreshold` 在 `onShow()` 时被重新设置
   ```typescript
   onShow() {
     const level = getMoyuLevel(_baseTotalMoney)
     _lastLevelThreshold = level.threshold  // ← 重新设置，覆盖上一次升级的记录
   }
   ```

2. **升级标志没有重置**：`_levelUpShowing` 在返回首页时仍为 `true`，导致后续升级被拦截
   ```typescript
   _showLevelUpModal(level: MoyuLevel) {
     if (_levelUpShowing) return  // ← 上次升级设置为 true，此次无法弹窗
     _levelUpShowing = true
     // ...
   }
   ```

3. **升级逻辑有缺陷**：只有当 `newLevel.threshold > _lastLevelThreshold` 时才弹窗，但如果用户返回首页重新加载，此判断会失效

### 修复方案

**步骤 1**：在 `onShow()` 首行重置升级标志
```typescript
onShow() {
  // V1.0.1 Fix: 重置升级弹窗标志，确保返回时能弹出新升级
  _levelUpShowing = false

  _settings = getSettings()
  // ... 其余逻辑 ...
}
```

**步骤 2**：改进升级检测逻辑
```typescript
_checkLevelUp(currentTotalMoney: number) {
  const newLevel = getMoyuLevel(currentTotalMoney)

  // 首次初始化时（_lastLevelThreshold === -1）
  if (_lastLevelThreshold < 0) {
    _lastLevelThreshold = newLevel.threshold
    return
  }

  // 检测到升级（新阈值 > 旧阈值）
  if (newLevel.threshold > _lastLevelThreshold) {
    _lastLevelThreshold = newLevel.threshold

    // 更新 UI
    this.setData({
      levelName: newLevel.name,
      levelEmoji: newLevel.emoji,
      levelColor: newLevel.color,
      isGoldLevel: newLevel.isGold,
    })

    // V2.2: 持久化升级信息（防止切页面时消失）
    setPendingLevelUp(newLevel)

    // 触发多感官反馈
    this._triggerCoinRain()
    this._showLevelUpModal(newLevel)
  }
}

_showLevelUpModal(level: MoyuLevel) {
  // V1.0.1 Fix: 检查升级弹窗标志
  if (_levelUpShowing) return
  _levelUpShowing = true

  if (_settings?.vibrateEnabled) {
    try {
      wx.vibrateShort({ type: 'medium' })
    } catch (_) {}
  }

  wx.showModal({
    title: '🎉 吗喽进化了！',
    content: `${level.emoji} ${level.name} 解锁！\n\n你的累计摸鱼收益突破了新门槛，\n继续摸，继续进化！`,
    confirmText: '去战报看看',
    cancelText: '继续摸鱼',
    success: (res) => {
      _levelUpShowing = false
      clearPendingLevelUp()
      if (res.confirm) wx.switchTab({ url: '/pages/stats/index' })
    },
    fail: () => { _levelUpShowing = false },
  })
}
```

**涉及文件**：
- `pages/index/index.ts` - 修改 `onShow()` 和 `_checkLevelUp()` 方法

## 问题 5：热力图方块数异常 + 当日未染色

### 现象
1. 热力图（Canvas）显示的方块数**不对**（应为 30 个，实际不清楚）
2. 当日开启摸鱼后，对应的热力图方块**仍为灰色**，应为绿色（表示有摸鱼数据）

### 根本原因

**问题 5a：方块数异常**
- 代码定义为 30 天 → 6×5 网格（30 个格子）✅
- 但 Canvas 绘制时可能遗漏了某些逻辑
- 或者 `buildHeatmapData()` 没有正确生成 30 个日期

**问题 5b：当日未染色**
- `buildHeatmapData()` 使用 `moyuDaysMap[dateStr]` 读取数据
- 但 `moyuDaysMap` 中的数据来自已提交的会话（`commitMoyuSession()`）
- **当日正在摸鱼时**，数据还未提交，所以 `moyuDaysMap[today]` 为 0
- 导致当日格子显示为灰色（level 0）

### 修复方案

**问题 5a 修复**：确保 `buildHeatmapData()` 正确生成 30 个格子

```typescript
function buildHeatmapData(moyuDaysMap: Record<string, number>): {
  rows: HeatmapCell[][]
  maxSeconds: number
} {
  const DAYS = 30
  const cells: HeatmapCell[] = []
  const today = new Date()

  let maxSeconds = 0

  // V1.0.1 Fix: 生成近 30 天的数据（从 29 天前开始，到今天）
  for (let i = DAYS - 1; i >= 0; i--) {
    const date = new Date(today)
    date.setDate(date.getDate() - i)
    const y = date.getFullYear()
    const mo = String(date.getMonth() + 1).padStart(2, '0')
    const da = String(date.getDate()).padStart(2, '0')
    const dateStr = `${y}-${mo}-${da}`
    const seconds = moyuDaysMap[dateStr] || 0
    if (seconds > maxSeconds) maxSeconds = seconds
    cells.push({ dateStr, seconds, level: 0 })
  }

  // 计算色阶（4 档）
  for (const cell of cells) {
    if (cell.seconds === 0) {
      cell.level = 0
    } else if (maxSeconds > 0) {
      const ratio = cell.seconds / maxSeconds
      cell.level = ratio < 0.25 ? 1 : ratio < 0.5 ? 2 : ratio < 0.75 ? 3 : 4
    }
  }

  // V1.0.1: 单行展示（30 个格子横向排列）
  const rows: HeatmapCell[][] = [cells]

  return { rows, maxSeconds }
}
```

**问题 5b 修复**：战报页加载时包含**当日未提交的摸鱼数据**

```typescript
_loadStats() {
```

## 问题 6：圆形仪表盘下方缺工时显示

### 现象
- 圆形进度条下方文字为倒计时："还有 8 小时 30 分钟下班"
- **缺少**：今日已工时 和 今日已摸工时 的显示
- 用户建议：应显示实际上班工时，便于了解工作进度

### 修复方案

在首页 `pages/index/index.ts` 中添加**工时显示逻辑**：

```typescript
// 在 data 中添加字段
data: {
  // ... 现有字段 ...
  todayWorkedTimeStr: '00:00:00',      // V1.0.1: 今日已工时
  todaySlackingTimeStr: '00:00:00',    // V1.0.1: 今日已摸工时
}

// 改进 _updateProgress() 方法
_updateProgress() {
  if (!_settings) return

  // 已工作秒数
  const workedSecs = calcTodayWorkedSeconds(_settings)

  // 计算总工时秒数（上班时间 - 休息时间）
  const [startH, startM] = _settings.workStartTime.split(':').map(Number)
  const [endH, endM] = _settings.workEndTime.split(':').map(Number)
  const lunchDuration = _settings.lunchBreakEnabled
    ? (parseInt(_settings.lunchBreakEnd.split(':')[0]) - parseInt(_settings.lunchBreakStart.split(':')[0])) * 3600 +
      (parseInt(_settings.lunchBreakEnd.split(':')[1]) - parseInt(_settings.lunchBreakStart.split(':')[1])) * 60
    : 0
  const totalWorkSeconds = (endH - startH) * 3600 + (endM - startM) * 60 - lunchDuration

  // 摸鱼秒数
  const slackingSecs = this.data.slackingSeconds

  // 计算百分比
  const workedPercent = Math.min(100, Math.round((workedSecs / totalWorkSeconds) * 100))
  const slackingPercent = Math.min(100, Math.round((slackingSecs / totalWorkSeconds) * 100))

  // 更新进度条样式：两层圆形（已工作绿色 + 已摸鱼蓝色）
  const style = `background: conic-gradient(
    #66BB6A ${workedPercent}%,
    #42A5F5 ${workedPercent}%,
    #42A5F5 ${workedPercent + slackingPercent}%,
    #C8E6C9 ${workedPercent + slackingPercent}%
  )`

  this.setData({
    progressPercent: workedPercent,
    slackingPercent,
    progressStyle: style,
    todayWorkedTimeStr: formatDuration(workedSecs),        // V1.0.1: 显示工时
    todaySlackingTimeStr: formatDuration(slackingSecs),    // V1.0.1: 显示摸鱼时
  })
}

// 在 _startClock() 中每秒更新
_startClock() {
  if (_clockTimer !== null) return
  _clockTimer = setInterval(() => {
    if (!_settings) return
    this.setData({
      todayWorkedEarnings: formatMoney(calcTodayWorkedEarnings(_settings)),
      monthEarnings: formatMoney(calcMonthEarnings(_settings)),
    })
    this._updateProgress()  // ← 包含工时更新
    this._updateTimeUntilOffWork()
  }, 1000)
}
```
