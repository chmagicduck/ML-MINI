## 三、数据模型（utils/types.ts）

### UserSettings（存储 key: `"userSettings"`）

```typescript
interface UserSettings {
  monthlySalary: number        // 月薪（元），0 表示未配置
  payDay: number               // 发薪日 1~31
  workStartTime: string        // "09:00"
  workEndTime: string          // "18:00"
  lunchBreakEnabled: boolean
  lunchBreakStart: string      // "12:00"
  lunchBreakEnd: string        // "13:00"
  eveningBreakEnabled: boolean
  eveningBreakStart: string    // "17:30"
  eveningBreakEnd: string      // "18:00"
  workdayMode: WorkdayMode     // 'double'|'sat-off'|'sun-off'|'big-small'
  joinDate: string             // "YYYY-MM-DD"
  retirementDate: string       // "YYYY-MM-DD"
  soundEnabled: boolean
}
```

**DEFAULT_SETTINGS**：`monthlySalary=0`（用于判断用户是否完成初始设置），其余字段均有合理默认值。`hasSettings = monthlySalary > 0`。

### WorkdayMode 语义

| key | 含义 |
|---|---|
| `double` | 周六、周日均休 |
| `sat-off` | 仅周六休（周日上班）|
| `sun-off` | 仅周日休（周六上班）|
| `big-small` | 奇数 ISO 周双休，偶数 ISO 周仅周日休 |

**重要约束**：大小周用 ISO 年度周数（`getISOWeekNumber`），不是月内第几周，保证跨月连续性。

### PoopStats（存储 key: `"poopStats"`）

```typescript
interface PoopStats {
  totalSeconds: number
  totalEarnings: number
  sessions: PoopSession[]      // 最多保留 200 条（slice(-200)）
}
interface PoopSession {
  date: string   // "YYYY-MM-DD"
  seconds: number
  earnings: number
}
```

## 五、计算引擎公式（utils/calculator.ts）

```
日薪     = monthlySalary / 当月工作日数
秒薪     = 日薪 / (getDailyWorkMinutes × 60)
今日摸鱼收益 = 秒薪 × slackingSeconds
本月已赚 = (本月已过工作日数 + 今日工时进度) × 日薪
```

**今日工时进度** = `workedMinutes / getDailyWorkMinutes`，其中 `workedMinutes` 实时扣除已过的午休/晚休分钟数（与分母保持一致）。

**发薪日计算**：目标日期钳位到该月实际最大天数（防 31 号在短月溢出）。

**核心函数列表**（均已导出）：

```
parseTimeToMinutes(timeStr) → number
getDailyWorkMinutes(settings) → number
getWorkingDaysInMonth(year, month, mode) → number
getWorkingDaysElapsed(settings) → number
getSecondSalary(settings) → number
calcTodayEarnings(settings, slackingSeconds) → number
calcMonthEarnings(settings) → number
getTodayWorkProgress(settings) → 0~1
getDaysToPayday(payDay) → number
getDaysToWeekend() → number（距最近周日）
getDaysToRetirement(retirementDate) → number
formatMoney(amount) → string（≥1万显示"X.XX万"，NaN/Infinity 返回"¥0.00"）
formatDuration(totalSeconds) → "HH:mm:ss"
```

## 六、首页计时器架构（pages/index/index.ts）

### 关键设计决策

1. **模块级变量**：`_timer`、`_settings`、`_audio` 均为模块级（非 Page.data），因 Page 是单例故安全。

2. **计时器精度**：用时间戳差值而非累加：
   ```typescript
   const baseSeconds = this.data.slackingSeconds
   const startAt = Date.now()
   setInterval(() => {
     const seconds = baseSeconds + (Date.now() - startAt) / 1000
   }, 100)
   ```

3. **音频单例**：`_audio` 在首次调用 `playCoinsSound()` 时创建，之后 `seek(0)` 复用，避免实例数超限。

4. **生命周期**：
   - `onLoad`：读系统信息（状态栏高度）、随机 slogan
   - `onShow`：重载设置 + 秒数 → 刷新数据 → **若 `isSlacking && _timer===null` 则重启计时器**（二级页返回关键修复）
   - `onHide` / `onUnload`：调 `_pause()`（停计时器 + 存秒数）

5. **`hasSettings` 判定**：`monthlySalary > 0`，否则显示引导 banner，但计算仍执行（值为 0）。

### Page.data 字段

```
statusBarHeight, progressStyle, progressPercent,
isSlacking, slackingSeconds, slackingTimeStr,
todayEarnings, monthEarnings, secondSalary,
daysToPayday, daysToWeekend, daysToRetirement,
hasSettings, slogan
```

### 卡片网格布局（关键约束）

**不能用** `flex-wrap + calc(50% - Xrpx)`，部分机型浮点舍入会导致单列。

**正确方案**：显式行容器 + `flex: 1`：
```xml
<view class="cards-container">         <!-- flex-direction: column -->
  <view class="cards-row">             <!-- flex-direction: row; gap: 20rpx -->
    <view class="card card-half">...</view>   <!-- flex: 1; min-width: 0 -->
    <view class="card card-half">...</view>
  </view>
</view>
```

### 进度环实现

三层叠加（`position: absolute`）：
1. `.ring-track`（300rpx 圆，背景色 `#C8E6C9`）
2. `.ring-progress`（同尺寸，`style` 注入 `conic-gradient(#66BB6A X%, #C8E6C9 X%)`）
3. `.ring-inner`（244rpx 内圆，白色，`top/left: 28rpx`）
