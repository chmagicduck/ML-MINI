# 优化方案（已确认版）

更新时间：2026-03-12
范围：任务1（清理无效/不可达代码）+ 任务2（重构全局存储与计算逻辑）

## 一、目标
1. 事件按行记录，支持后续情报中心报表和手动编辑。
2. 累计收益按事件发生时薪资快照入账，不随用户后续涨薪回算。
3. 修复前后台切回后“鱼额宝偷偷帮你从老板兜里掏走x”的计算口径。
4. 修复个人基地总收益和等级进度不实时、改薪资后累加异常问题。
5. 清理皮肤系统及其他无效/不可达代码。

## 二、核心设计
### 1) 单一事实来源
- 以 `moyuStats.events` 作为事实数据。
- 每条事件记录：`id/dayKey/startAt/endAt/seconds/money/source/note`。
- `totalMoney/totalSeconds/moyuDaysMap` 由事件聚合得到并持久化缓存。

### 2) 收益口径
- 每次提交增量时，同时写入 `seconds` 和 `money`。
- `money` 由“事件发生当时”的秒薪计算得到。
- 因此涨薪后不会重算历史累计收益。

### 3) 离线收益弹窗口径
- `app.onHide` 仅记录隐藏时间戳。
- `index.onShow` 按区间 `[hiddenAt, now]` 计算有效工作秒数（扣除非工作时段/休息时段）。
- 弹窗金额 = 区间有效工作秒数 × 当前秒薪。

### 4) 个人基地实时刷新
- `profile` 页面进入后启动 1s 轻量刷新。
- 页面隐藏/卸载即停止刷新。
- 总收益和等级进度按统一口径实时更新。

## 三、清理清单（任务1）
1. 皮肤系统全部移除：
- `types.ts` 中皮肤类型与常量。
- `storage.ts` 中 `getCurrentSkin/setCurrentSkin`。
- `profile` 里“外观皮肤”入口及相关存储重置项。

2. 无效广告位代码移除：
- `settings/basic.ts` 中 `adUnitId`。
- `settings/basic.wxml` 中永不展示的广告块。

3. 重复与冗余逻辑收敛：
- 统一摸鱼存储读写口径与事件聚合口径。

## 四、落地顺序
1. 调整数据结构与存储函数（事件模型 + 聚合缓存）。
2. 首页计时提交流程改为事件行写入（含时间区间）。
3. 离线收益弹窗改为区间计算。
4. 个人基地实时刷新与统一口径计算。
5. 清理皮肤/广告等无效代码。
6. 编译回归与手工验证。

## 五、验收标准
1. 每次摸鱼提交后都能形成事件行。
2. 后台切回弹窗金额与区间有效工时一致。
3. 个人基地在停留页面时收益和等级实时刷新。
4. 调整薪资后：历史累计收益不被回算，新增收益按新薪资入账。
5. 工程内不存在皮肤系统和无效广告位相关代码。

1. 小程序未上线：不引入历史版本包袱，直接使用当前数据结构。
2. 默认视为初始版本结构。
